Introducción¶

Hearthstone es un juego de cartas coleccionables creado por la empresa Blizzard [1]. Este juego ha visto hasta 6.5 millones de jugadores concurrentes y debido a su popularidad sitios como Hearthpwn.com [2] fueron desarrollados para permitirle a los jugadores una herramienta para crear y compartir mazos de prueba.

Heartpwn nos ha encargado con la bsqueda de factores que permitan clasificar mazos de cartas en categoras que no estaban definidas dentro de las diferentes plataformas que sern futuramente absorbidas por el sitio. Esto con el objetivo de clasificar los datos no cubiertos por ellas como los nuevos formatos de juegos implementados que son estndar y salvaje.

En este reporte se analizarn resultados relacionados a la clasificacin de mazos a travs de las categoras provistas por parte de Heartpwn para asistir en la introduccin de datos de mazos externos a su plataforma. El objetivo final siendo el desarrollo de uno o ms modelos experimentales que puedan asistir a esta tarea.

En el conjunto de datos se encuentran registrados distintos mazos subidos a la plataforma de Heartpwn desde el ao 2013 hasta 2017 y fueron proporcionados por el sitio Kaggle [3].

Limpieza de dataset¶

Se realiza una limpieza de las variables no relevantes, con grandes cantidades de variables vacas, incorrectas o faltantes. Lo primero que se realiz fue evaluar las variables que no tenan relevancia en el contexto planteado, dejando solo las variables deseadas, posteriormente se limpiaron dichas variables dejando nicamente las que ofrecan informacin relevante para el contexto.

Varaibles no relevantes :

  • craft_cost
  • deck_set
  • title
  • user
  • rating
  • deck_id

Se tiene en la mira la variable con datos faltantes, errneos o nulos

  • deck_archetype

Tenemos que verificar si esta variable puede ser relevante para la propuesta experimental, en caso de ser afirmativo, entonces se tiene pensado limpiar los datos dejando solo los valores que son conocidos y eliminar los datos desconocidos de dicha variable.

El dataset actualmente contiene datos desde el ao 2013, esto no es algo favorable para lo que necesitamos, ya que en ese momento el juego estaba en fase "beta", por lo tanto las cartas y decks no son relevantes para el anlisis final. Por lo que se eliminara los datos que sean anterior al 11 de marzo del 2014. Siendo esta la fecha de salida del juego para todo pblico.

Preguntas y Problemas¶

  1. Problema de clasificacin binaria basado en la variable "deck_format".

  2. Encontrar una caracterstica o atributos de un mazo que puedan realizar una mejor clasificacin de un formato.

  3. Cules seran los atributos que puedan permitir hacer una mejor prediccin del formato de un mazo?

Propuesta Experimental¶

  • Como sabemos que un mazo se compone de atributos que hacen referencia a cartas del mazo, vamos a agrupar cartas que se juegan en un formato y en el otro para as lograr hacer una clasificacin de un mazo respecto a las cartas que utiliza, para aquello utilizaremos una tcnica de clustering.

  • Vamos a clasificar con 3 modelos rbol de decisin, regresin logstica y naive bayes para identificar y comparar modelos sirve para clasificar el formato de un mazo.

  • Analizaremos la correlacin de los atributos no relacionados a las cartas de un mazo con el formato de este para as detectar cules son los atributos nos permiten hacer una mejor clasificacin.

def predecir_formato_arbol(decks_df, card_columnas):

    X = decks_df[card_columnas]
    y = decks_df['deck_format']

    X_train, X_test, y_train, y_test = train_test_split(X, y)
    
    # Arbol de decision
    clf = tree.DecisionTreeClassifier()
    clf.fit(X_train, y_train)
    pred = clf.predict(X_test)
    
    print(classification_report(y_test, pred))
    print('Matriz de confusin: \n', metrics.confusion_matrix(y_test, pred))
    
    return clf

def predecir_formato_bayes(decks_df, card_columnas):

    X = decks_df[card_columnas]
    y = decks_df['deck_format']

    X_train, X_test, y_train, y_test = train_test_split(X, y, test_size=0.5, random_state=0)
    
    # Naive Bayes
    clf = GaussianNB()
    clf.fit(X_train, y_train)
    pred = clf.predict(X_test)
    
    print(classification_report(y_test, pred))
    print('Matriz de confusin: \n', metrics.confusion_matrix(y_test, pred))
    return clf
